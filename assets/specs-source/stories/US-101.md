# US-101 — Creare un task

## User Story
Come membro del progetto voglio creare un task con titolo, descrizione e
assegnatario opzionale.

## Acceptance Criteria
- `POST /api/v1/tasks` accetta body JSON:
  - `project_id` (UUID, obbligatorio)
  - `title` (stringa 3..200 char, obbligatorio)
  - `description` (stringa 0..5000 char, **opzionale** — default stringa vuota)
  - `assignee_id` (UUID, opzionale)
- Validazione completa lato server: rifiuto 400 con `error.code`
  machine-readable in caso di violazione.
- Response 201 con `data.task` completo (id, created_at, status="open").

## Note
Il campo `description` è opzionale: in passato (BUG-501) la mancata
descrizione causava NPE perché il serializer assumeva non-null. Ora il
default lato API è stringa vuota; lato dominio è `Optional[str]`.
