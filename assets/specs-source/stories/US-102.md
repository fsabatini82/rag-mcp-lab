# US-102 — Listare task del progetto con filtri

## User Story

Come **membro di un progetto** voglio **elencare i task applicando filtri
opzionali** così da **focalizzarmi solo su ciò che mi riguarda**.

## Acceptance Criteria

- `GET /api/v1/tasks` accetta:
  - `project_id` (UUID, **obbligatorio**)
  - `status` (enum: `open` | `in_progress` | `done` | `archived`, opzionale)
  - `assignee_id` (UUID **oppure** stringa `"null"` per "non assegnati", opzionale)
  - `label` (stringa, opzionale)
  - `page` (int ≥ 1, default 1), `page_size` (int ∈ [1, 100], default 25)
- Distinzione **param assente vs. param esplicitamente nullo** è obbligatoria:
  - parametro `assignee_id` non passato → nessun filtro su assegnatario
  - parametro `assignee_id=null` → filtra task con `assignee_id IS NULL`
  - parametro `assignee_id=<uuid>` → filtra task con `assignee_id = <uuid>`
- Ordine risultati: `updated_at DESC, id ASC`.
- Errori:
  - `400 BAD_REQUEST` se `project_id` mancante / malformato.
  - `403 FORBIDDEN` se l'utente non appartiene al progetto.
- Response shape: `{ data: Task[], meta: { page, page_size, total } }`
  (vedi ADR-002).

## Scenari BDD

```gherkin
Scenario: Filtro per status open
  Given un progetto P con 5 task (3 open, 2 done)
  When chiamo GET /api/v1/tasks?project_id=P&status=open
  Then ricevo 200 con data.length == 3
  And ogni task ha status "open"
  And l'ordine è updated_at DESC

Scenario: Filtro per assignee non specificato
  Given un progetto P con 4 task: 2 assegnati, 2 non assegnati
  When chiamo GET /api/v1/tasks?project_id=P
  Then ricevo 200 con data.length == 4

Scenario: Filtro esplicito per assegnatario nullo
  Given un progetto P con 4 task: 2 assegnati, 2 non assegnati
  When chiamo GET /api/v1/tasks?project_id=P&assignee_id=null
  Then ricevo 200 con data.length == 2
  And ogni task ha assignee_id == null
```

## Note implementative
- Trattare con cautela il caso `assignee_id=null`: in passato (BUG-502) il
  filtro veniva ignorato perché il binding consumava il valore stringa
  prima del cast a UUID. Vedi anche ADR-002 sul pattern Sentinel.
